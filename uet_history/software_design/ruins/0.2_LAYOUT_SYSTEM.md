# 🧩 **BLUEPRINT 0.2 — LAYOUT SYSTEM (ฉบับสมบูรณ์)**

_“Layout = กรอบฟังก์ชันทั้งหมดของ UI, ทุกหน้าในระบบอยู่ภายใต้กฎเดียวกัน”_

# 0) **เป้าหมายของ Layout System**

Layout System ถูกสร้างขึ้นเพื่อ:

### 1. ทำให้ทุกหน้าในแพลตฟอร์ม UET ใช้ pattern เดียวกัน

→ ลดความซับซ้อน  
→ ทำให้ต่อยอดง่าย  
→ Dev ทุกคนเข้าใจตรงกัน

### 2. สามารถปรับเปลี่ยน/ออกแบบ Panel ได้โดยไม่แตะโครงสร้างใหญ่

→ รองรับการขยายแบบ Social Media, Research Platform, หรือ Data Platform

### 3. สร้าง “ภาษาออกแบบ” กลาง

→ Source อยู่ซ้าย  
→ Chat ตรงกลาง  
→ Studio/Detail อยู่ขวา  
→ Drawer ซ่อน  
→ Modal เด้ง  
→ Layout = แผ่นพื้นใหญ่

# 1) **องค์ประกอบหลักของ Layout System**

UET Layout System มี 6 กลุ่มใหญ่:

|Element|อธิบายสั้น ๆ|อยู่ในทุกหน้า?|
|---|---|---|
|**Page**|ส่วนที่เปลี่ยนตาม URL|✔|
|**Layout**|โครงร่างพื้นฐาน เช่น 3 คอลัมน์|✔|
|**Panel**|พื้นที่ย่อยที่ทำงานเฉพาะเรื่อง|✔|
|**Drawer**|ลิ้นชักสไลด์ซ้าย/ขวา|✖ (เฉพาะบางหน้า)|
|**Modal**|หน้าต่างเด้งทับทั้งหมด|✖|
|**Shell**|Navigation + Sidebar|✔|

ทั้งหมดนี้เป็น “ภาษาหลัก” ของแพลตฟอร์ม

# 2) **UET Standard Layout (3-Panel System)**

> โครงสร้างหลักของทุกหน้าในแพลตฟอร์ม

```
┌───────────────────────────────────────────────┐
│                   NAVBAR                      │
└───────┬───────────────────────────┬──────────┘
        │                           │
        │                           │
┌───────▼───────┐       ┌───────────▼───────────┐
│   LEFT PANEL   │       │       MAIN PANEL      │
│ (Source/Feed)  │       │   (Chat / Content)    │
└───────┬────────┘       └───────────┬──────────┘
        │                            │
┌───────▼────────┐       ┌───────────▼──────────┐
│  RIGHT PANEL    │       │       OPTIONAL        │
│   (Studio/Info) │       │    EXPAND ZONE        │
└─────────────────┘       └──────────────────────┘
```

### **Left = Source / Navigation / Knowledge**

ที่อยู่ของข้อมูลทั้งหมด

### **Center = Interaction / Chat / Content**

ที่อยู่ของการทำงานหลักทุกอย่าง

### **Right = Studio / Detail / Autoprompt**

ที่อยู่ของการเขียน, วิเคราะห์, เอกสาร, Draft

# 3) **Rules — กฎเหล็กของ Layout System**

## 3.1 Panel ห้ามทับหน้าที่กัน

หมายความว่า:

- Source = เฉพาะข้อมูล
- Chat = เฉพาะสื่อสาร
- Studio = เฉพาะเอกสาร/งานเขียน
- Metrics = เฉพาะตัวเลข

→ เพื่อไม่ให้สับสนและเพื่อให้ AI Agent ทำงานได้ง่ายขึ้น

---
## 3.2 ทุก Panel มี State ของมันเอง

ตัวอย่าง:
- Source Panel จดจำว่าเปิดไฟล์ไหนอยู่
- Chat Panel จดจำว่า session ไหน
- Studio Panel จดจำว่ากำลังแก้ markdown ไหน

→ แต่ทั้งหมด Sync กับ backend ผ่าน State Engine

---
## 3.3 Layout ต้อง “Responsive แบบมีกฎ”

มือถือ → แสดงทีละ panel  
แท็บเล็ต → ซ้าย + กลาง  
คอมพิวเตอร์ → ซ้าย + กลาง + ขวา

---
# 4) **ประเภท Panel ทั้งหมดในระบบ**

## 4.1 **SOURCE PANEL**

ตำแหน่ง: ซ้าย  
หน้าที่:

- File Browser
- Graph View
- KB Navigator
- Project Structure
- Search

ฟีเจอร์เฉพาะ:
- แสดงความสัมพันธ์แบบ Obsidian Graph
- Open/Edit File
- Multi-KB Switch

---
## 4.2 **MAIN PANEL (CHAT / CONTENT)**

ตำแหน่ง: กลาง  
หน้าที่: interaction หลักทั้งหมด

- Private Chat
- Project Chat
- Community Chat
- Document Reading
- Dashboard View

ฟีเจอร์เฉพาะ:
- AI Context Overlay
- Inline Citation
- Multi-Chat Session

---
## 4.3 **STUDIO PANEL**

ตำแหน่ง: ขวา  
หน้าที่:

- Note Editor (Markdown)
- Autoprompt Tools
    
- Drafting
    
- Report Generation
    
- Python Sandbox
    

ฟีเจอร์เฉพาะ:

- Auto-Save
    
- AI Assist (Fill → Expand → Validate)
    
- Notebook-like system
    

---

## 4.4 **FEED PANEL**

ตำแหน่ง: Center (เฉพาะหน้า Home/Community)

หน้าที่:

- Scrollable Feed
    
- Post
    
- Pin
    
- Voting
    

---

## 4.5 **METRICS PANEL**

ตำแหน่ง: Center หรือ Right (ตาม context)

หน้าที่:

- Balanced Scorecard
    
- KPI Tracker
    
- Wallet Ledger
    

---

## 4.6 **PROFILE PANEL**

Drawer ขวา

หน้าที่:

- User Info
    
- Settings
    
- Subscription
    
- Token Usage
    

---

# 5) **DRAWER SYSTEM**

ใช้เมื่อ:

- ต้องการความลึกข้อมูล
    
- ต้องการ pop-out ที่ไม่รบกวน layout
    

ตัวอย่าง:

- รายชื่อสมาชิกใน Project
    
- รายละเอียดเงินบริจาค
    
- การตั้งค่า session
    

---

# 6) **MODAL SYSTEM**

ใช้เมื่อ:

- ต้องการบังคับ Action
    
- ต้องการยืนยัน
    
- ต้องการ Input สำคัญ
    

ตัวอย่าง:

- Login
    
- Create Project
    
- Delete File
    
- Publish Knowledge
    

---

# 7) **GLOBAL SHELL**

ประกอบด้วย:

- Navbar: Home / Theory / Project / Community / Donate / Login
    
- Sidebar: History / Profile / Workspace Switch
    
- Hotkeys:
    
    - **⌘ + K** Command Palette
        
    - **⌘ + /** Global Search
