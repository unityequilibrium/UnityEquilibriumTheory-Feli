# # 🟩 1. Core System Terms (ศัพท์ระบบหลัก)

แต่ละคำประกอบด้วย 3 ส่วน:  
**ความหมาย (Meaning)** + **ใช้ในระบบยังไง (How it’s used)** + **ตัวอย่างจริง (Example)**

### **1. Core Engine**

**Meaning:**  
	กลไกหลักของระบบทั้งหมด (ส่วนที่สั่งงาน AI, RAG, KB, memory, sync)
**How it's used:**  
	ใช้คอนโทรลว่าเวลาผู้ใช้สั่งคำสั่งอะไร ระบบดึงข้อมูลจากไหน, คิดยังไง, เก็บยังไง
**Example:**  
	“เวลาแชทใน Project ระบบจะใช้ Core Engine + Project KB เท่านั้น”

---
### **2. Global Knowledge Base (Global KB)**

**Meaning:**  
	แหล่งความรู้ใหญ่ของระบบ ทั้งหมดรวมกันทั้งโปรเจกต์
**How it's used:**  
	ใช้ตอนแชทส่วนตัว (Private Chat) หรือคุยเรื่องใหญ่ ๆ เกี่ยวกับระบบ
**Example:**  
	เวลาเราถาม “สรุป UET Theory คืออะไร” → AI ใช้ Global KB ตอบ

---
### **3. Project Knowledge Base (Project KB)**

**Meaning:**  
	สมองเล็กเฉพาะแต่ละโปรเจกต์ แยกไฟล์ แยกเวคเตอร์ ไม่ปนกัน
**How it's used:**  
	ใช้ตอนแชทในหน้าโปรเจกต์หรือเขียนเอกสารของโปรเจกต์นั้น
**Example:**  
	เราเปิด Project: Canal Robot → AI ใช้ข้อมูลเฉพาะโปรเจกต์นี้

---
### **4. Vector Store**

**Meaning:**  
	ที่เก็บ “ความหมายของข้อความ” ในรูปแบบเวกเตอร์
**How it's used:**  
	ตอน AI ต้องหาไฟล์ที่คล้ายกัน, หาเนื้อหา, หาความสัมพันธ์ของไฟล์
**Example:**  
	เราพิมพ์ว่า “ขอดูโน้ตเรื่อง KPI ของโปรเจกต์นี้” → ระบบไปค้น vector

---
# # 🟧 2. UI Layout Terms (ศัพท์สำหรับหน้าจอ/โครงสร้าง)

### **5. Page**

**Meaning:**  
หน้าใหญ่ที่มี URL เป็นของตัวเอง เช่น /projects /theory /home

**How it's used:**  
ใช้แบ่ง “พื้นที่” ในเว็บ

**Example:**  
หน้า Projects จะมี layout ซ้าย-กลาง-ขวา

---
### **6. Layout**

**Meaning:**  
โครงสร้างการแบ่งพื้นที่ในหน้า เช่น ซ้าย-กลาง-ขวา

**How it's used:**  
กำหนดว่าหน้านี้มี Panel อะไรบ้าง

**Example:**  
หน้า Project มี layout:  
Left = Source  
Middle = Chat  
Right = Studio

---
### **7. Panel**

**Meaning:**  
แผง (ช่องใหญ่) ที่แสดงข้อมูลหรือทำงานเฉพาะอย่าง

**How it's used:**  
ทุกหน้าใช้ Panel เพื่อแสดงเนื้อหา เช่น Source / Chat / Studio / Feed

**Example:**  
Panel Source = บริหารไฟล์  
Panel Chat = พูดคุย AI  
Panel Studio = สร้าง/แก้โน้ต

---
### **8. Drawer**

**Meaning:**  
แถบเลื่อนออกมาด้านข้างแบบทับหน้าจอ

**How it's used:**  
ใช้สำหรับข้อมูลเสริม เช่น Member list, Project settings

**Example:**  
คลิกปุ่ม "Members" → Drawer ลื่นออกมาทางขวา

---
### **9. Modal**

**Meaning:**  
หน้าต่างเด้งขึ้นกลางจอ บังกว่า 80%

**How it's used:**  
ใช้ตอนต้องใส่ข้อมูลสำคัญมาก เช่น Login, Rename Project

**Example:**  
กด “New Project” → modal เด้งขึ้นให้พิมพ์ชื่อโปรเจกต์

---

# # 🟪 3. Interaction Terms (ศัพท์เกี่ยวกับการทำงานร่วมกัน)

### **10. Feed**

**Meaning:**  
โซนไหลลงเหมือน Social Media / Home Feed ที่ใช้แสดงข้อมูลอัปเดต
**How it's used:**  
ใช้ในหน้า “Community Feed” หรือ “Theory Feed”
**Example:**  
เลื่อนดูโพสต์ทฤษฎีใหม่ ๆ หรือข้อมูลคนใช้งาน

---

### **11. KPI Wallet**

**Meaning:**  
คอนเซ็ปต์ “กระเป๋า KPI” ที่ใช้ติดตามเป้าหมายของแต่ละโปรเจกต์/ผู้ใช้

**How it's used:**  
ใช้ในการวัดผลแบบ Balanced Scorecard

**Example:**  
Project Robot มี KPI 10 เป้าหมาย → แสดงใน Panel Wallet

---

### **12. Studio Actions**

**Meaning:**  
ชุดคำสั่งที่ทำได้ใน Studio เช่น summarize / rewrite / create note

**How it's used:**  
เอาไว้ทำงานรวดเร็วใน Panel Studio

**Example:**  
คลิก “Summarize” → AI ประมวลผลไฟล์ markdown ปัจจุบัน

---

---

# # 🟩 4. RAG/AI Terms (ศัพท์เกี่ยวกับระบบ AI)

### **13. RAG (Retrieval-Augmented Generation)**

**Meaning:**  
เทคนิคให้ AI “อ่านไฟล์จริงก่อนตอบ”

**How it's used:**  
ใช้ทั้ง Global KB และ Project KB

**Example:**  
ตอนสรุปไฟล์รายงาน → AI ดึงไฟล์จริงมาคิดก่อนสร้างคำตอบ

---

### **14. Memory Sync**

**Meaning:**  
ระบบจำว่า “ผู้ใช้พูดอะไร” แล้ว sync ไปที่ KB ถ้าจำเป็น

**How it's used:**  
ใช้ตอนเขียนโน้ตอัตโนมัติ

**Example:**  
เราพิมพ์: “จำไว้ว่ากระบวนการ A คือขั้นตอนใหม่” → AI sync เข้าระบบ

---

# # 🟦 5. File/Content Terms (ศัพท์สำหรับไฟล์และเนื้อหา)

### **15. Markdown Node**

**Meaning:**  
ไฟล์ markdown ที่เป็น “บล็อคความรู้” หนึ่งบล็อค

**How it's used:**  
เป็นหน่วยความรู้พื้นฐานของระบบ

**Example:**  
Theory_1.md, ProjectSpec.md, KPIReport.md

---

### **16. Canvas (อนาคต)**

**Meaning:**  
พื้นที่วางบล็อคโน้ตแบบอิสระเหมือน Obsidian Canvas

**How it's used:**  
ใช้สร้าง mindmap / flow / research diagram

**Example:**  
เชื่อม Note: Concept → Problem → Solution ด้วยเส้น
