# 🧭 **BLUEPRINT 0.3 — PAGE_DEFINITIONS (ฉบับสมบูรณ์)**

_“Page = หน่วยการนำเสนอ + หน่วยฟังก์ชัน ที่ถูกกำหนดหน้าที่ชัดเจน”_  
ทุก Page จะมี:

- หน้าที่หลัก
    
- โครงสร้าง Layout
    
- Panel ที่ใช้
    
- ฟังก์ชันของหน้า
    
- กฎพฤติกรรม (Behavior Rules)
    
- Event ที่เกี่ยวข้อง
    
- API ที่ใช้
    
- State ที่ต้องเก็บ
    

ทั้งหมดนี้เพื่อให้ AI/Dev สามารถ Generate UI + Logic ได้ทันทีโดยไม่ต้องเดา

---

# 🔥 **0) UET Platform — หน้าที่ของ Page System**

- ทำให้ทุก URL = 1 Responsibility
    
- ลดสับสน
    
- ลด logic ทับซ้อน
    
- ช่วย AI Generate Code ได้ถูกต้อง
    
- ช่วย Dev ไม่ต้องคิดซ้ำ
    
- ช่วย UX ทำงานบน pattern เดียวกัน
    

---

# 🧱 **2) Page Template — แม่แบบหลักของทุกหน้า**

ทุก Page จะใช้ template เดียวกัน:

```
PAGE_TEMPLATE = {
  layout: "STANDARD_3_PANEL",
  panels: {
    left: SOURCE | NAV | FILTER,
    main: CONTENT | CHAT | FEED,
    right: STUDIO | DETAIL | TOOL
  },
  drawers: [],
  modals: [],
  state: [],
  api: [],
  events: []
}
```

# ⭐ PAGE 1: **HOME PAGE**

### 📌 URL

`/`

### 🎯 หน้าที่

- แนะนำระบบ
    
- แสดง Feed (ประกาศ, อัพเดต, โปรเจกต์เด่น)
    
- จุดเริ่มต้นที่เข้าใจง่าย
    

### 🧱 Layout

- Left = Navigation
    
- Main = Home Feed
    
- Right = “Quick Start Panel”
    

### 🧩 Panel

- **Main Panel → Feed Panel**
    
- **Right Panel → Quick Start**  
    (ปุ่มเริ่มแชท, อ่านทฤษฎี, เข้าร่วมโปรเจกต์)
    

### ⚙ ฟังก์ชันหลัก

- Scroll Feed
    
- เปิดบทความ
    
- เปิด Project
    
- เริ่ม Chat

`/theory`

- AI Summarize
    
- อ้างอิงแบบ Chat-with-Document

# ⭐ PAGE 4: **PROJECT DETAIL PAGE (หัวใจของแพลตฟอร์ม)**

### URL

`/projects/:id`

### หน้าที่

- ทำงานวิจัย/สร้างผลงาน
    
- Chat + Files + Studio รวมกัน
    
- เป็น workspace หลักแบบ Obsidian x Notion x AI Agent Fusion
    

### Layout (FIXED)

- **Left = Source Panel (ไฟล์โปรเจกต์ + Graph View)**
    
- **Main = Chat Panel / Content Panel (เลือกได้)**
    
- **Right = Studio Panel**
    

นี่คือหน้าเดียวที่ใช้ “Full Workspace Engine”

### Sub-tabs ของ MAIN PANEL

- Chat
    
- Document
    
- Dashboard
    
- Running Agents
    
- KPI / Wallet
    

### ฟังก์ชันหลัก

- อัปโหลดไฟล์
    
- Autoprompt
    
- Run Agent
    
- Generate Report
    
- แก้ไขไฟล์ Markdown
    
- ทำกราฟความรู้
    
- คำนวณ KPI
    
- วิเคราะห์ข้อมูลด้วย Python
    

> **นี่คือ Page ใหญ่ที่สุด และจะเป็นฐานของการสเกลแพลตฟอร์ม**
